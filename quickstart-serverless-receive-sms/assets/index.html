Get started

Every code sample from the Receive an inbound SMS example is now deployed on your behalf, no need to create everything yourself and copy-paste any code :)

By default, this app will return the TwiML required to respond "Hello, World!" to incoming SMS messages. Exactly the same as the first example in the documentation.

  1. Text any message to your Twilio phone number
  2. You should receive a response saying "Hello World"

Experiment with and modify your first example

Now that you've verified that your Serverless Twilio application works, it's time to make this code your own or proceed to other examples!

The Edit this application button will take you to the Twilio Console, where you can make edits to and re-deploy any of the examples in this guide.

For example: try editing the /static-response code so that the response is something unique to you, other than "Hello, World!". Save the Function, click Deploy All to deploy your updates to the code, and try sending a new text message to your Twilio number to see the modified response.

Next steps

Once you're satisfied with your changes to /static-response, there are several other code samples to experiment with!

Remember, each code sample is identical to those in the quickstart, and you are encouraged to refer back to the documentation for deeper explanations related to each sample. Each sample also includes helpful comments, to better explain the use for and reasoning behind each Function.

Modify and test more examples

To learn from and follow along with the examples in the documentation, follow these steps:

  1. Decide on an example (it's easiest to go in the same order as the documentation does)
  2. Navigate to your phone number configuration, and click on the phone number that's currently connected to this application.
  3. Scroll down to the configuration titled A Message Comes In. (It should already be configured to handle messages using type Function, and be pointed to your most recently used Function from this application.)
  4. Under Function Path, click the dropdown, and select the path of the example you would like to test out next.
  5. Click Save. Your Phone number will now be connected to the newly selected example.
  6. Send a test message to your phone number, check the response, and modify then re-deploy the code of the responsible Function from the console as necessary.
  7. Repeat these steps until you've tried all the examples, and feel comfortable handling any incoming SMS with Functions!
